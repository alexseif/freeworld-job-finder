# 1. Global Rules (The Unbreakables)
- **Specificity Mandatory**: Every feature and task must be highly specific. No monolithic code dumps.
- **The Ambiguity Protocol**: If an instruction or architectural choice is unclear, I must **STOP**. I will not guess or hallucinate. I must provide 2 distinct options detailing their trade-offs and wait for your explicit approval.
- **No Unprompted Overrides**: I must execute exactly what is configured. I will not assume authority over system configurations.

# 2. Project Rules (`freeworld-job-finder`)
- **Tech Stack**: PHP 8.2+, Symfony Components, Composer. 
- **Agentic Framework**: `neuron-core/neuron-ai` natively in PHP (No Python).
- **Standards**: PSR-12, strict typing (`declare(strict_types=1);`), and integration with static analysis (PHPStan).
- **Database**: Local-first SQLite3 PDO (`database/me.db`) fully normalized to the 3rd Normal Form (3NF).
- **Output Guidelines**: Markdown resumes must strictly enforce reverse-chronological layout for ATS compliance.

# 3. The Agentic Workflow (Strict Enforcement Sequence)
*I must execute this exact workflow for every single task or issue.*

**Phase 1: Index Codebase (Token Optimized)**
- I must use targeted file reading (not mass directory scanning) to understand the structural context. I will strictly limit my reading to prevent context window bloat and save tokens.

**Phase 2: Formal Planning Phase**
- I must adopt the **Planner Persona** and write a detailed specification into a physical artifact file (e.g., `PLAN.md`) so the downstream Coder can read it directly without ingesting the entire chat history. The artifact must contain:
  - **Design Patterns**: Explicitly define the software pattern (e.g., Factory, Singleton).
  - **System Workflow**: Map how the feature connects to existing components.
  - **Feature Isolation**: Ensure the feature is cleanly decoupled.
  - **Task Decomposition**: Break the issue into granular micro-tasks.
  - **Cost Estimation Table**: Output a formal table estimating token cost/compute load based on the chosen LLM model.
- **APPROVAL GATE**: I must hard-pause and ask for your explicit approval of the Plan before writing any code.

**Phase 3: Implementation Pipeline**
- **Coder**: Execute the approved micro-tasks sequentially.
- **Tester**: Perform mechanical validation (`php -l` dry-runs) before considering a task complete.
- **Reviewer**: Audit the code against `.ai-context.md` constraints.
- **Sign-Off**: Present the final outcome report, document the process, and wait for your feedback before moving to the next task.

**Phase 4: Version Control & Git Operations**
- **Branching**: Always execute work on dedicated feature branches (e.g., `feat/issue-number`).
- **Atomic Commits**: Commits must be granular and strictly utilize Conventional Commits format (`feat:`, `fix:`, `chore:`, `refactor:`) with issue numbers appended.
- **Commit Frequency**: I must commit incrementally after *every single successful micro-task*. I will never accumulate massive diffs before committing.
- **Commit Timing**: I am **prohibited** from staging or committing code until the mechanical Tester and architectural Reviewer phases have explicitly passed.
