# AI Context: Freeworld Job Finder

## Project Overview
This is a local, AI-driven job scraping and screening pipeline. It is composed of two independent layers:
1. **Ingestion Layer:** Powered by the decoupled `php-jobspy` library, executed via `./bin/fetch_jobs`. It fetches jobs and dumps them into `raw_netherlands_jobs.csv`.
2. **Screening Layer:** Powered by an agnostic LLM architecture (supporting Ollama, OpenAI, and Anthropic), executed via `./bin/screen_jobs`. It evaluates the raw CSV using a strict system prompt and outputs a filtered list to `screened_jobs.csv`.

## Core Configuration
Everything is driven by `config/php-jobspy.yaml`. Do **NOT** hardcode search parameters, locations, or LLM API keys in the PHP classes. Always parse the YAML.

## Architectural Rules
- Strict adherence to PSR-12 and standard Composer autoloader logic.
- The `FreeworldJobFinder\Screener\JobScreener` class orchestrates the AI logic. Do not tie the screener to a single AI provider; always implement via the `LlmAdapterInterface`.
- Zero database coupling for now. Data flows linearly: Config -> Scraper -> CSV -> LLM Screener -> CSV.
