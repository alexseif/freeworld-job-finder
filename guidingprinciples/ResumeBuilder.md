# Resume Builder Architecture

## Separation of Concerns

The Resume Builder is strictly an **output generation compiler**. It does not query the database, nor does it perform LLM inference. It operates on a single principle: **Take a validated Data Transfer Object (DTO) and map it to a strictly formatted Markdown template.**

### 1. The Data Layer (Input DTOs)
The local inference engine (`me.db` + Llama) prepares a structured, finalized JSON payload. The Builder receives this payload blindly.

### 2. The Compiler Engine (Logic)
The compiler applies layout rules based on geopolitical constraints (e.g., masking location/dates for EU GDPR, compressing layout for US 1-Page).

### 3. The Presentation Layer (Output)
A clean, flat, ATS-optimized Markdown document.

---

## The Core Fields (DTO Schema)

To keep the builder simple and graceful, the input payload must adhere to these exact fields:

### Contact Information (`ContactInfo`)
*   `fullName`: String
*   `targetTitle`: String (Dynamically generated by Llama to mirror the vacancy)
*   `contact`: Object
    *   `email`: String
    *   `phone`: String
    *   `location`: String (Masked or revealed based on geopolitical format)
    *   `links`: Array `[{ platform: 'LinkedIn', url: '...' }, { platform: 'Portfolio', url: '...' }]`

### Professional Summary (`Summary`)
*   `content`: String (A dense 3-4 sentence paragraph synthesized by Llama, explicitly targeting the vacancy's core pain points)

### Technical Skills (`Skills`)
*   `categories`: Array of Objects
    *   `name`: String (e.g., "Backend", "Infrastructure")
    *   `skills`: Array of Strings (Only skills present in the vacancy *and* verified in `me.db` are injected here)

### Work Experience (`Experience`)
*   `roles`: Array of Objects (Strictly reverse-chronological)
    *   `company`: String
    *   `title`: String
    *   `startDate`: String (Format: YYYY-MM)
    *   `endDate`: String (Format: YYYY-MM or 'Present')
    *   `location`: String
    *   `highlights`: Array of Strings (Max 3-5 bullets. These are contextually selected by Llama from the master database to exactly match the target vacancy's requirements)

### Education (`Education`)
*   `degrees`: Array of Objects
    *   `institution`: String
    *   `credential`: String
    *   `year`: String (Can be suppressed based on age-bias masking rules)
